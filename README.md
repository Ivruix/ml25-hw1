# ДЗ 1
Streamlit приложение тут: https://ml25-hw1.streamlit.app

## Что сделал

1. Предобработка данных
   - Убрал пропуски (заполнил медианами)
   - Удалил дубликаты
   - Преобразовал признаки в числа (убрал единицы измерения)

2. Работа с признаками
   - Удалил столбец name
   - Закодировал категориальные признаки (One-Hot Encoding)
   - Стандартизировал числовые признаки

3. Построение моделей
   - Обучил модели: Linear Regression, Lasso, ElasticNet, Ridge
   - Подобрал лучшие параметры через GridSearchCV
   - Оценил качество по R2 и MSE

4. Создание севрвиса
   - Сохранил лучшую модель (Ridge) в pickle
   - Сделал Streamlit приложение

## Результаты

Качество моделей (R2):
- Все модели: ~0.59
- Ridge с категориальными фичами: 0.64

Бизнес-метрика:
- Только 25% прогнозов в пределах 10% от реальной цены

**Вывод:** Модели объясняют больше половины изменчивости цен, но точность для бизнеса не очень.

## Что помогло улучшить качество

Добавление категориальных признаков дало 0.05 к R2.

Признаки типа топлива, коробки передач и продавца оказались важными для прогноза цены.

## Что не получилось

1. Регуляризация не улучшила качество
